## Erkenntnisse

1. **Der semantische Benchmark differenziert Modelle** — im Gegensatz zum alten Router-Katalog (640/640 deterministic pass in Sekunden).
2. **Router-Merge-Bug:** Bei erzwungener Semantik darf ein deterministischer Tool-Rebound die LLM-Rolle nicht überschreiben. Fix in CaiLama: `prefer_semantic` in `_merge_semantic_with_deterministic`.
3. **Task-Katalog zu starr für plausible Alternativen:** `profile_game` + Training ist fachlich vertretbar, scheitert aber an `expected_step_sequence`.
4. **Fallback-Rate Task (~17 %):** JSON-Parse oder leere Schritte → `planning_source=fallback`; Prompt/Striktur weiter schärfen.
5. **Feedback-Agent:** Bewertet `routing_decision`/`task_plan`-JSON; Vertragsfehler aus `error_status` werden mit Score 1/1 auto-bewertet.
